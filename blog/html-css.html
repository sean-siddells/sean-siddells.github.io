What are the differences between relative, absolute and fixed positioning?

Positioning in CSS is used as a way of moving html element placements on a web page. An "element" is just a basic unit of HTML that has an opening and closing tag that relates to it's content. For example, a "<h1>" tag signifies that the contents between this and the closing tag "</h1>" will be a heading element.

Every element is a block of pixels on a webpage, and many variables of these blocks can be adjusted through CSS like size, height/width settings and position. The position property allows you to change the position of the element based on several different principles.

Relative:
The relative position changes the position of the element in relation to where it would usually go. This means that elements will be displayed in the order they are written (either from top to bottom or left to right). So, using the same heading element from the example above, the CSS "position: relative;" property, combined with an attribute like "right: 50px;" will move the element 50 pixels from the right of its usual position on the page. Absolute:
The absolute position changes the position of the element relative to the parent element that has the position: "relative" or "absolute". If there is no parent element that fits these requirements, it will position itself relative to the page. Absolute positioned elements are taken out of the normal flow of the document, and therefore can overlap other elements or cause spacing issues. So if we used "position: absolute;" on the the first example and apply the attribute "right: 50px;", it will move the element 50px from the right of the page itself. Note that the spacing isn't consistent between the bottom of the heading and the next paragraph unlike prior examples.

Fixed:
The fixed position adjusts the position of the element in relation to the viewport ie. the computer display or phone screen that the web page is being viewed on. This means that the element will stay visible to the viewer even when the page is scrolled up or down. This is great if the element is an essential component to interacting with the website, however this means that this element may overlap another. The position attributes in this case will be used to position the element in the web browser. In this example, I have used the word "Hello" which should be floating in the bottom left corner. The attributes applied to the heading look like this:

.fixed {
position: fixed;
bottom: 0px;
left:0px;
margin: 0px
}
